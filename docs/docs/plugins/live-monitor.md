# 直播监控

主动监控 B 站直播间开播/关播状态并发送通知，无需依赖 blrec 等外部工具。

:::info 2.0 配置方式
监控映射、轮询间隔、WebSocket 开关等均在 **Web Admin → 直播监控** 与 **设置** 中配置，不再使用 `LIVE_MONITOR_*` 环境变量。
:::

## 功能特点

- **主动监控**：定时轮询 + WebSocket 弹幕双重机制
- **自动通知**：检测到开播/关播时自动推送
- **多房间 / 多群组**：每个房间可配置多个目标群/好友
- **@全体成员**：开播时支持 @全体（需机器人管理员权限）

## 监控原理

### 1. WebSocket 弹幕监听（主要，默认启用）

连接 B 站直播间弹幕 WebSocket，实时监听状态命令：

- `LIVE` → 开播（秒级响应）
- `PREPARING` → 关播（秒级响应）

### 2. API 轮询（备用）

定时调用 `getInfoByRoom` 接口：

- WebSocket 启用时作为备用，间隔较长（约 5 分钟）
- WebSocket 禁用时作为主要方式

### 直播状态

| 状态 | 值 | 说明 |
|------|-----|------|
| PREPARING | 0 | 未开播 |
| LIVE | 1 | 直播中 |
| ROUND | 2 | 轮播中 |

## 命令

| 命令 | 别名 | 说明 |
|------|------|------|
| `/直播状态 [房间号]` | `/查直播`、`/live` | 查询指定房间直播状态 |
| `/监控列表` | `/直播监控列表` | 列出当前群监控的房间 |

### 示例

```
用户: /直播状态 12345678
机器人: 🔴 主播名称
       状态：直播中
       房间号：12345678
       标题：今天来玩游戏！
       人气：12345
       直播间：https://live.bilibili.com/12345678
```

## 通知格式

开播通知示例：

```
🎉 主播名称 开播啦！
直播间标题：今天来玩游戏！
房间号：12345678
点我直达：https://live.bilibili.com/12345678
```

文案可在 **消息模板** 页面自定义。

## 注意事项

1. 请求过于频繁可能被 B 站限制，检查间隔不建议低于 30 秒
2. 配置 B 站账号可提高 API 与 WebSocket 稳定性
3. @全体需要机器人具有群管理员权限
4. 首次启动会初始化各房间状态，不会发送通知
